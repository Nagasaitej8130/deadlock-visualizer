Deadlock in Operating Systems

Definition

A deadlock is a situation where a set of processes are permanently blocked because each process is waiting for a resource held by another process.

Basic Idea

Deadlock occurs due to a circular waiting condition. Each process holds a resource and waits for another resource that is already held by another process.

Example

Process P1 holds Resource R1 and waits for R2. Process P2 holds R2 and waits for R1. Neither process can proceed, creating a deadlock.

Four Necessary Conditions

  • Mutual Exclusion: Resources cannot be shared.
  • Hold and Wait: Processes hold resources while waiting for others.
  • No Preemption: Resources cannot be forcibly taken.
  • Circular Wait: Processes form a cycle of dependencies.

Graph Representation (RAG)

  • P → R : Request edge (process is waiting)
  • R → P : Allocation edge (resource is assigned)

Key Insight

A cycle in the graph indicates deadlock only when multiple processes are involved. A self-loop (single process) is not a deadlock.
